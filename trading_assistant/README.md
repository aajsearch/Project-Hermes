# Trading Assistant (Alpaca Paper)

A **paper-trading bot** for Alpaca that trades **leveraged ETFs (equities)** and **options calls** (one position per underlying). It runs in a loop, uses market-hours gating, persists state to avoid repeat buys and manage exits, and supports risk controls (stop loss, take profit, max hold, cooldowns, and optional portfolio-level profit lock / loss limit).

**Default safety:** Paper mode only, kill switch on, no execution when market is closed (unless overridden for testing).

---

## What it does

- **Equities:** For each symbol in the watchlist, fetches bars, runs a trend + RSI strategy (`lev_etf_trend_v1`), and places market BUY/SELL per profile (notional, stop/take profit, trail, max hold, cooldown).
- **Options:** For each underlying, uses the same underlying trend signal; on BUY selects a call (DTE/delta/IV/spread filters) and places a market order; on SELL or stop/take profit/max hold exits the option.
- **State:** Persisted in `state/state.json` (positions, pending orders, cooldowns, option recos, portfolio cooldown, MFE/MAE tracker, buys_last_hour).
- **Reconciliation:** On startup and every cycle, state is reconciled with the broker (broker is source of truth) so restarts do not cause duplicate buys.
- **Ledger:** Every fill is appended to `logs/trades.csv`; signals to `logs/signals.csv`.
- **EOD report:** `python scripts/generate_eod_report.py [YYYY-MM-DD]` produces `logs/daily_report_YYYY-MM-DD.md`.
- **Dashboard:** Streamlit UI at `ui/app.py` for viewing trades, KPIs, and state (see [Usage](docs/usage.md)).

## Quickstart (paper mode)

1. **Environment**
   ```bash
   # Create .env with ALPACA_API_KEY and ALPACA_SECRET_KEY (or copy from .env.example if present)
   pip install -r requirements.txt
   ```
2. **Config**  
   Ensure `config/settings.py` has `KILL_SWITCH = True` and `config/watchlist.yaml` lists your symbols/underlyings and profiles.
3. **Run**
   ```bash
   python main.py
   ```
   The bot connects to Alpaca **paper**, logs buying power, runs startup reconciliation, then loops: when market is open it runs a cycle every 5 minutes; when closed it logs "Market closed. Waiting..." and sleeps until the next 5-minute boundary.

## How to run

- **Normal (loop):** `python main.py` — runs until you stop it (Ctrl+C).
- **Single cycle (test):** In `config/settings.py` set `RUN_ONCE = True`; then `python main.py` runs one cycle and exits.
- **EOD report:** `python scripts/generate_eod_report.py` (today) or `python scripts/generate_eod_report.py 2026-02-09`.
- **Streamlit UI:** From project root, `python3 -m streamlit run ui/app.py` (see [Usage](docs/usage.md)).

---

## How to stop

- **Interactive:** Ctrl+C in the terminal where `main.py` is running.
- **PM2 (if used):** `pm2 stop trading_assistant` (see [Usage](docs/usage.md)).

## Safety defaults

- **Paper only:** The code uses `make_trading_client(paper=True)` in `main.py`; no live trading unless you change that and use live keys (not recommended).
- **Kill switch:** `config/settings.py` has `KILL_SWITCH = True`. If set to `False`, the bot computes signals but does **not** place any order (logs "SKIP: kill_switch_off").
- **Market closed:** Execution is gated by `is_market_open_now()` (Pacific) and `ALLOW_EXECUTION_WHEN_CLOSED = False` / `ALLOW_OPTION_EXECUTION_WHEN_CLOSED = False`, so no orders when the market is closed unless you override for testing.

## How to safely test changes

- **Signals only, no orders:** Set `KILL_SWITCH = False` in `config/settings.py`. Run `python main.py`; you should see signals in logs and "SKIP: kill_switch_off", with no orders placed. *Source: `core/engine.py` — `can_place_orders` requires `KILL_SWITCH is True`.*
- **No execution (equities/options still computed):** Set `EXECUTE_TRADES = False` and/or `EXECUTE_OPTION_TRADES = False`. The bot still runs strategies and appends to signals but does not submit orders. *Source: `core/engine.py` — `can_execute` / `can_execute_opt`.*
- **Single cycle then exit:** Set `RUN_ONCE = True` in `config/settings.py`. Run `python main.py`; it runs one cycle (reconcile, signals, and if market open and kill switch on, execution) then exits. *Source: `main.py` — after `run_cycle()`, if `RUN_ONCE` then return.*
- **After-hours test run:** Set `FORCE_RUN_WHEN_MARKET_CLOSED = True` and `ALLOW_EXECUTION_WHEN_CLOSED = True` (and optionally `ALLOW_OPTION_EXECUTION_WHEN_CLOSED = True`). Run with `RUN_ONCE = True` to do one cycle using last available data; orders may stay "accepted" until next open. Revert these before live use. *Source: `main.py`, `core/engine.py`.*
- **Shadow mode:** Not implemented. A future idea would be to log "would BUY/SELL" without placing orders (e.g. a dedicated flag or dry-run mode).

## Source of truth (key files)

- Entry point: `main.py` — `main()`
- Loop and cycle: `main.py` (while True, `run_cycle`), `core/scheduler.py` — `is_market_open_now()`, `sleep_until_next_5min()`
- Execution and risk: `core/engine.py` — `run_cycle()`, `_exit_all_positions()`, portfolio check, `can_place_orders`
- State: `core/state_store.py` — `load_state()`, `save_state()`, position/cooldown/MFE/MAE/buys_last_hour helpers
- Ledger: `core/ledger.py` — `append_trade()`, `TRADES_LEDGER_CSV`
- Orders: `broker/alpaca/orders.py` — `submit_market_order()`, `wait_for_order()`

## What can go wrong

- **Repeated buys after restart:** Usually fixed by reconciliation; if state was deleted or broker has positions not in watchlist, ensure watchlist matches broker and restart once. See [Troubleshooting](docs/troubleshooting.md).
- **UUID not JSON serializable:** Order IDs are stored as strings (`str(o.id)` in `broker/alpaca/orders.py`); state uses `JSONEncoder` for UUID in `core/state_store.py`. If you see serialization errors, check that no raw UUID is written to state.
- **Market closed spam:** Normal when market is closed; the bot logs "Market closed. Waiting..." each wake. Reduce log level or cadence if desired (cadence is currently fixed at 5 min in code).

## How to verify

- **Paper + reconcile:** Run `python main.py`; log should show "Paper account buying_power=...", "Startup reconcile complete.", then either cycle output or "Market closed. Waiting...".
- **Kill switch:** Set `KILL_SWITCH = False`, run one cycle with market open or `FORCE_RUN_WHEN_MARKET_CLOSED=True`; log should contain "SKIP: kill_switch_off".
- **EOD report:** After a day with trades, run `python scripts/generate_eod_report.py 2026-02-09` and confirm `logs/daily_report_2026-02-09.md` exists and has summary/exit reasons/trades.
